雪花算法

长情 发布于 2024-04-06

最后更新于 2025-04-06

雪花算法（Snowflake）是一种分布式 ID 生成算法，由 Twitter 开发，能生成全局唯一的 64 位长整型 ID，通常用于分布式系统。

雪花算法生成的 ID 组成

雪花算法生成的 ID 由 64 位（64-bit）组成，一般结构如下：

符号位 时间戳 数据中心 ID 机器 ID 序列号
1 bit 41 bit 5 bit 5 bit 12 bit

详细解析

  1. 1 bit（符号位）
    • 最高位为 0，因为 Snowflake 生成的 ID 是正数。
  2. 41 bit（时间戳）
    • 记录时间戳，通常是当前时间与一个起始时间（epoch）之间的毫秒差值。
    • 可用时间：2^41 / (1000 * 60 * 60 * 24 * 365) ≈ 69 年。
  3. 5 bit（数据中心 ID）
    • 用于区分不同的数据中心，最多支持 2^5 = 32 个数据中心。
  4. 5 bit（机器 ID）
    • 用于区分同一数据中心的不同机器，最多支持 2^5 = 32 台机器。
  5. 12 bit（序列号）
    • 同一毫秒内的自增序列，最多支持 2^12 = 4096 个不同 ID。
    • 当同一毫秒内超过 4096 个请求时，会等待下一毫秒。

雪花算法的特点

✅ 高效：每秒最多可生成 32 * 32 * 4096 = 4,194,304 个唯一 ID。
✅ 趋势递增：ID 按时间递增，适合数据库索引优化。
✅ 分布式支持：支持多个数据中心和机器并行生成。
❌ 时钟回拨问题：服务器时钟回拨可能导致 ID 生成重复，需要额外处理。

适用场景

  • 分布式系统唯一 ID 生成（如订单号、用户 ID）
  • 高并发系统（如微博、日志系统）
  • 数据库分表（可用于主键，避免自增 ID 的冲突）

在 Java 中可以使用 MyBatis-Plus 提供的雪花 ID 或 Hutool 的 Snowflake 生成唯一 ID，避免手动实现复杂逻辑。
